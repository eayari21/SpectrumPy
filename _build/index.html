Getting Started¶

    1. To get started, launch the app (MacOS) or executable (Windows).
    2. A file dialog should show up, prompting the user for the location of the trace files they want to view.
    3. Included with the SpectrumPY installation should be a "traces" directory containing well-tested Hyperdust examples. NOTE: the user must specify the folder containing the trace files, not one of the trace files themselves.
    4. Once a trace folder has been chosen, SpectrumPY automatically detects the number of channels and respective shots, and reads them in.
    5. After reading in the traces, the main interactive plot window should display 4 channels by default.
    6. If this was not the dataset you were after, navigate to the upper left menu and select File, Import Data, Import Scope Data (.trc), and go back to step 2.
    7. If you would like to export the data to a .csv file, navigate to the upper left menu and select File, Export Data, Export to .csv. Then, find "SpecOutput.csv" in the same directory that SpectrumPY is installed in.
    8. To change the trace file (or accelerator shot) that you're viewing, navigate to the upper left menu and select View, Choose Trace. A drop down list of all the auto detected traces will appear for the user to select. If for some reason the TOF channel is not displaying, please reimport your scope data. This issue is being looked into.
    9. If you would like to alter the number of channels being displayed or which channels are being displayed, select View, Change Channels. This will bring up the external channel choosing window, where the user can select 1-4 channels for display. The order the checkboxes are selected is the order that the plots will be displayed in the main window. When finished selecting your channels, press submit plot and close the window. If the wrong number of channels is entered, the plot will revert to its default settings.
    10. If you get stuck at any point, navigate to the upper left menu and select Help, then "Read the Docs" to bring up these html pages.
    11. If there any bugs are encountered, please reach out the Ethan.Ayari@lasp.colorado.edu.
